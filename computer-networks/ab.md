# КОМПЬЮТЕРНЫЕ СЕТИ: СИСТЕМАТИЧЕСКИЙ КОНСПЕКТ

## 1. Базовый понятийный аппарат

**Сеть (network)** — множество узлов, связанных отношениями взаимодействия.

**Компьютерная сеть (computer network)** — совокупность вычислительных машин, телекоммуникационного оборудования и специализированного ПО, обеспечивающая обмен данными между абонентами.

**Узел (node)** — любая сетевая единица (хост, коммутатор, маршрутизатор). Конечные системы (хосты) соединяются линиями связи через коммутаторы пакетов.

**Пакет (packet)** — квант данных, снабженный служебными заголовками, передаваемый как неделимая единица в сетях с коммутацией пакетов.

**Трафик (traffic)** — агрегированный поток пакетов.

---

## 2. Телекоммуникационный контекст

**Телекоммуникации** (греч. *tele* — удаленно + лат. *communicatio* — связь) — передача/прием информации любого типа на расстояние посредством электромагнитных сигналов.

**Телекоммуникационная сеть** — техническая инфраструктура, включающая три функциональных домена:

| Домен | Функция |
|-------|---------|
| Сети доступа (access) | подключение конечного оборудования пользователей |
| Магистраль (core/backbone) | агрегация и транзит трафика |
| Центры управления сервисами (SCP/DC) | обработка данных, сервисная логика |

**Типология по природе трафика**:
- компьютерные сети (дата-трафик)
- телефонные сети (голос)
- радиосети (широковещательная аудиосвязь)
- телеграфные сети (текстовые сообщения)
- телевизионные сети (видеопоток)

## 3. Классификация сетей

### 3.1 По территориальному охвату

| Тип | Аббр. | Диапазон |
|-----|-------|----------|
| Локальная | LAN | помещение, здание, кампус |
| Городская | MAN | район, город |
| Глобальная | WAN | страна, континент, планета |

### 3.2 По масштабу организации

- сети рабочих групп (workgroup)
- сети отделов (departmental)
- кампусные сети (campus)
- корпоративные сети (enterprise)

### 3.3 По методу коммутации

| Метод | Принцип |
|-------|---------|
| Коммутация каналов | фиксация ресурсов на сессию |
| Коммутация пакетов | статистическое мультиплексирование |
| Коммутация сообщений | хранение-и-пересылка (store-and-forward) |

### 3.4 По типу физической среды

- проводные (guided): витая пара, коаксиал, оптоволокно
- беспроводные (unguided): радиоканалы, ИК-диапазон, спутник

### 3.5 По организации управления

- одноранговые (peer-to-peer)
- клиент-серверные (client-server)

## 4. Целевая функция компьютерных сетей

**Разделяемые ресурсы (network share)** — устройства или данные с удаленным доступом.

**Услуги доступа:**
- удаленный вход (remote login)
- передача файлов (file transfer)
- удаленный вызов процедур (RPC)
- совместное использование устройств

**Услуги коммуникации:**
- электронная почта (1:1)
- списки рассылки (1:N)
- телеконференции (N:N)
- видеоконференции

## 5. Архитектурные модели

В основе любой сетевой коммуникации лежат уровневые модели. Они стандартизируют взаимодействие, разделяя задачи по уровням — от передачи битов до прикладных сервисов.

### 5.1 Модель OSI (ISO/IEC 7498)

7 уровней (снизу вверх):

| Уровень | Функция |
|---------|---------|
| 1. Физический | биты, сигналы, разъемы |
| 2. Канальный | кадры, MAC-адреса, коммутация |
| 3. Сетевой | пакеты, маршрутизация, IP-адреса |
| 4. Транспортный | сегменты, надежность (TCP/UDP) |
| 5. Сеансовый | управление диалогом, сессии |
| 6. Представления | кодирование, шифрование, сжатие |
| 7. Прикладной | протоколы приложений (HTTP, FTP, SMTP) |

### 5.2 Модель TCP/IP (DoD)

4 уровня (функционально соответствует OSI):

| Уровень | Соответствие OSI | Протоколы |
|---------|------------------|-----------|
| Прикладной | 5–7 | HTTP, FTP, SMTP, SSH, DNS |
| Транспортный | 4 | TCP, UDP |
| Интернет | 3 | IPv4, IPv6, ICMP |
| Сетевых интерфейсов | 1–2 | Ethernet, Wi-Fi, PPP |

> Модель TCP/IP — рабочая основа современного Интернета. OSI — теоретический эталон для понимания и проектирования.

## 6. Базовые топологии соединений

Топология определяет структуру связей между узлами. Выбор влияет на отказоустойчивость, стоимость и пропускную способность.

| Топология | Характеристика | Применение |
|-----------|----------------|-------------|
| Шина (bus) | общая среда передачи, коллизии | устаревший Ethernet (10BASE2/5) |
| Звезда (star) | центральный узел (коммутатор) | доминирует в современных LAN |
| Кольцо (ring) | детерминированный доступ, отказоустойчивость (FDDI) | промышленные сети |
| Полносвязная (mesh) | каждый с каждым, избыточность | магистрали, дата-центры |
| Дерево (tree) | иерархическая звезда | кампусные и корпоративные сети |

---

## 7. Ключевые протоколы (нижние уровни)

Справочное ядро для понимания того, как данные физически перемещаются:

| Уровень (TCP/IP) | Протоколы | Назначение |
|------------------|-----------|------------|
| Сетевых интерфейсов | Ethernet (IEEE 802.3) | проводной доступ, кадры L2 |
| Сетевых интерфейсов | Wi-Fi (IEEE 802.11) | беспроводной доступ |
| Интернет | IPv4 / IPv6 | адресация и маршрутизация |
| Интернет | ICMP | диагностика (ping, traceroute) |
| Транспортный | TCP | надежная доставка с подтверждением |
| Транспортный | UDP | быстрая доставка без подтверждения |

## 8. Историческая хронология

| Период | Событие |
|--------|---------|
| 1960‑е | Проекты DARPA по объединению мейнфреймов; эксперименты с пакетной коммутацией |
| 1969 | Запуск ARPANET — объединение суперкомпьютеров оборонных и научных центров США |
| 1970‑е | Разработка TCP/IP (Vint Cerf, Bob Kahn) |
| 1980‑е | Переход ARPANET на TCP/IP; появление Ethernet |
| 1990‑е | Коммерциализация Интернета, World Wide Web |

> ARPANET — прямой предшественник Интернета. Все ключевые архитектурные решения (пакетная коммутация, сквозной принцип, TCP/IP) были отработаны в этом проекте.
