# Практическая работа №16  
## Групповые политики Windows 10

**Цель работы:**  
Изучить возможности редактора локальной групповой политики (gpedit.msc) в Windows 10, научиться настраивать различные политики для управления интерфейсом и поведением системы.

---

## Теоретическая часть

### 1. Запуск редактора локальной групповой политики
1. Нажмите **Пуск**.
2. Введите `gpedit.msc`.
3. Нажмите **Enter**.

<img width="473" height="333" alt="browser_Rw4UcVporj" src="https://github.com/user-attachments/assets/e1a555e6-f221-4a84-a36d-ed1b5f3dc59a" />

### 2. Структура редактора
Левая панель содержит две основные категории:
- **Конфигурация компьютера** — политики применяются ко всем пользователям, действуют до входа в систему.
- **Конфигурация пользователя** — политики применяются к текущему пользователю после входа.

Правая панель отображает политики выбранной категории.

### 3. Настройка политики
- Двойной щелчок по политике открывает окно настройки.
- Состояния: **Не задано**, **Включить**, **Отключить**.
- Некоторые политики требуют дополнительных параметров (пути к папкам, числа и т.д.).

### 4. Фильтрация политик
Чтобы быстро найти нужную политику:
1. Выберите ветвь **Конфигурация пользователя → Административные шаблоны → Все параметры**.
2. В меню **Действие** → **Параметры фильтра**.
3. Укажите ключевое слово, область поиска (заголовок, текст справки).
4. Включите фильтр кнопкой **Фильтр** на панели инструментов.

**Цель:** скрыть ненужные кнопки в окне, вызываемом `Ctrl+Alt+Del`.

**Выполнение:**
1. Открыть `gpedit.msc`.
2. Перейти в раздел:

   - **«Запретить блокировку компьютера»**

### Задание 2. Настройка панели мест (Places Bar) в диалогах «Открыть/Сохранить как»
**Цель:** добавить собственные папки в левую панель стандартных диалоговых окон.

**Выполнение:**
1. Открыть `gpedit.msc`.
2. Перейти в раздел:  
   `Конфигурация пользователя → Административные шаблоны → Компоненты Windows → Проводник Windows → Общее диалоговое окно открытия файлов`.
3. Дважды щёлкнуть политику **«Элементы, отображаемые в панели мест»**.
4. Выбрать **Включить**.
5. В полях **Элемент 1** — **Элемент 5** ввести пути к нужным папкам, например:
   - `C:\Users\Public\Documents`
   - `D:\Projects`
   - `\\server\share`

6. Нажать **ОК**, закрыть редактор.
7. Открыть любое приложение (например, Блокнот) → **Файл → Открыть** и убедиться, что в левой панели появились новые значки.

### Задание 3. Увеличение списка недавних документов в меню «Пуск»
**Цель:** изменить количество отображаемых недавних документов (по умолчанию 15).

**Выполнение:**
1. Открыть `gpedit.msc`.
2. Перейти в раздел:  
   `Конфигурация пользователя → Административные шаблоны → Компоненты Windows → Проводник Windows`.
3. Дважды щёлкнуть политику **«Максимальная длина списка "Недавние документы"»**.
4. Выбрать **Включить**.
5. Установить нужное значение (например, 30) в поле счётчика.
6. Нажать **ОК**.
7. Перезагрузить проводник или выйти из системы.  
   Проверить: **Пуск → Недавние документы** (если пункт отображается) – список стал длиннее.

### Задание 4. Включение диалога слежения за завершением работы
**Цель:** активировать окно, позволяющее указывать причину перезагрузки или выключения компьютера.

**Выполнение:**
1. Открыть `gpedit.msc`.
2. Перейти в раздел:  
   `Конфигурация компьютера → Административные шаблоны → Система`.
3. Дважды щёлкнуть политику **«Отображать диалог слежения за завершением работы»**.
4. Выбрать **Включить**.
5. В списке выбрать **Всегда**.
6. Нажать **ОК**.
7. Нажать **Пуск → Завершение работы**. Должно появиться окно с выбором причины.

## Контрольные вопросы

### 1. Что такое групповые политики?
**Групповые политики** — это набор правил и параметров, которые управляют рабочей средой пользователей и компьютеров в ОС Windows. Они позволяют централизованно или локально настраивать ограничения, права, интерфейс, параметры безопасности и многие другие аспекты системы.

### 2. Категории объектов групповых политик?
В локальном редакторе групповых политик (`gpedit.msc`) объекты делятся на две основные категории:
- **Конфигурация компьютера** — применяется ко всем пользователям данного ПК и действует при загрузке системы.
- **Конфигурация пользователя** — применяется к конкретному пользователю после его входа в систему.

### 3. Разделы групповых политик?
Внутри каждой категории политики сгруппированы по разделам, например:
- **Программное обеспечение** (установка/удаление приложений)
- **Параметры Windows** (безопасность, сценарии входа/выхода)
- **Административные шаблоны** (основная масса политик, разделённых по компонентам: система, сеть, проводник Windows и т.д.)

### 4. Запуск групповых политик?
Для запуска **локального** редактора групповых политик необходимо:
- Нажать **Win + R**, ввести `gpedit.msc` и нажать Enter.
- Либо через **Пуск** → в строке поиска ввести `gpedit.msc` и выбрать соответствующее приложение.

> **Примечание:** Редактор локальной групповой политики доступен только в версиях Windows 10 Pro, Enterprise и Education. В домашней версии (Home) он отсутствует.
